# PDCA評価: Time Block Optimizer

## Iteration 1

### Plan: アイデア具体化

**アイデア名:** TimeFlow - AI時間最適化アシスタント

**問題定義:**
- タスクが多すぎて優先順位がつけられない
- カレンダーにタスクを配置する作業自体が時間の無駄
- 予定通りに進まず、常にリスケが必要
- 理想と現実のギャップで罪悪感を感じる

**ターゲットユーザー:**
- フリーランス・個人事業主
- プロジェクトマネージャー
- 複数プロジェクトを抱えるエンジニア・デザイナー
- 時間管理に悩むビジネスパーソン全般

**解決策:**
1. タスクを投げ込むだけでAIが最適スケジュールを生成
2. エネルギーレベル（朝型/夜型）を考慮した配置
3. 実績データから精度を自動改善
4. リアルタイムで再最適化（遅延時の自動調整）
5. 「今日は何をすべきか」が一目瞭然

**成功指標:**
- DAU/MAU > 0.4（高頻度利用）
- タスク完了率が導入前比+30%
- スケジュール作成時間を80%削減
- ユーザー満足度スコア > 8.0/10

### Do: 評価スコアリング

#### PUGEF (潜在ユーザー × 利用頻度 × 緊急性)
**スコア: 8.5/10**

| 要素 | 評価 |
|------|------|
| 潜在ユーザー | 9/10 - 全ビジネスパーソンが対象（数億人規模） |
| 利用頻度 | 9/10 - 毎日使う（朝の計画、途中での調整） |
| 緊急性 | 8/10 - 時間管理の悩みは深刻だが生死には関わらない |

**理由:** 時間管理は普遍的な課題。毎日使うツールとして定着する可能性が高い。

#### ICE (Impact × Confidence × Ease)
**スコア: 7.5/10**

| 要素 | 評価 |
|------|------|
| Impact | 9/10 - 生産性向上は人生の質を大きく改善 |
| Confidence | 7/10 - AI推定精度次第だが、学習で改善可能 |
| Ease | 7/10 - カレンダーAPI、AI統合は実現可能レベル |

**理由:** 技術的には実現可能。ユーザー行動データの蓄積が成功のカギ。

#### Market (市場規模 × 成長性 × 参入障壁)
**スコア: 7.0/10**

| 要素 | 評価 |
|------|------|
| 市場規模 | 8/10 - 生産性ツール市場は巨大（Calendar/TODO統合） |
| 成長性 | 8/10 - リモートワーク増加で時間管理ニーズ拡大 |
| 参入障壁 | 6/10 - 既存強豪多数（Motion, Reclaim.ai など） |

**理由:** 市場は大きいが競合も多い。差別化が重要。

#### Advantage (技術的優位性 × 実現可能性 × 差別化)
**スコア: 7.0/10**

| 要素 | 評価 |
|------|------|
| 技術的優位性 | 7/10 - 最適化アルゴリズム + AI推定で差別化可能 |
| 実現可能性 | 8/10 - 既存技術で実装可能 |
| 差別化度 | 7/10 - UIデザインと推定精度で差別化 |

**理由:** 技術的には実現可能。UXと精度で勝負。

### 総合スコア: **7.5/10**

平均: (8.5 + 7.5 + 7.0 + 7.0) / 4 = **7.5**

### Check: ギャップ分析

**強み:**
- 普遍的な課題へのアプローチ
- 高頻度利用の可能性
- 技術的実現可能性

**弱点と改善優先度:**
1. **市場競合** (Market 7.0) - 既存プレイヤーとの差別化が不明確
2. **Confidence** (ICE 7.0) - AI推定精度の実証が必要

**改善アクション:**
1. 競合分析を深掘り → 明確な差別化軸を定義
2. 初期ユーザー体験（オンボーディング）を極限まで磨く
3. 「初回から実用レベルの推定」を目指す（過去データ不要）

### Act: 改善版アイデア

**差別化ポイントの強化:**

1. **「Zero-Config」スタート**
   - 過去データ不要、初日から使える
   - 一般的なタスクタイプ（執筆、コーディング、会議準備）の所要時間を事前学習済み
   - 3日使えば個人最適化完了

2. **「Energy-Aware」スケジューリング**
   - 時間帯別のパフォーマンスを学習
   - 「朝は頭脳労働、午後はルーチン」など自動配置
   - 既存ツールにない独自機能

3. **「Reality Check」モード**
   - 理想スケジュールと現実の乖離を可視化
   - 「この量は物理的に無理です」と警告
   - 無理なコミットメントを防ぐ

4. **超シンプルUI**
   - Notion風のミニマルデザイン
   - AI感を消した自然な操作感
   - 「カレンダー」ではなく「今日の流れ」として表示

## 最終評価

スコア 7.5/10 は十分に高く、プロトタイプ作成に値する。
差別化ポイントを明確にすることで市場競争力を持てる。
