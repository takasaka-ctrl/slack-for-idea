# PDCA評価: "Omni-Process Shadow (OPS)"

## 1. Plan: アイデア概要
ブラウザ操作エージェント（OpenAI Operator等）が「何をしたか」を可視化・監査し、かつ「次に何をすべきか」を人間がレビューするためのダッシュボード。

- **解決する課題:** エージェントのブラックボックス化（勝手に注文した、誤った情報を入力した等）への不安。
- **ターゲット:** エージェントを業務導入したいが、ガバナンスが懸念されるチーム。
- **コア機能:** 
  - エージェントの操作履歴（DOMイベント単位）のタイムライン表示。
  - 「承認待ち」ステップでの人間による介入。
  - 操作ログからの「ベストプラクティス」抽出。

## 2. Do: 市場性・技術実現性・独自性の評価

### 評価基準 (1-10)
1.  **市場ニーズ (Market Fit):** 9/10 (エージェント普及に伴い、必ず監査ツールが必要になる)
2.  **技術実現性 (Feasibility):** 8/10 (Playwright/Chrome DevTools Protocol経由でイベントキャプチャ可能)
3.  **独自性 (Uniqueness):** 7/10 (単なるロギングではなく「承認ワークフロー」を統合)
4.  **収益性 (Profitability):** 8/10 (B2B SaaSモデル、セキュリティ予算から捻出可能)

**総合スコア: 8.0/10** (基準値7.5をクリア)

## 3. Check: 懸念点
- **ブラウザ側の制約:** 拡張機能やプロキシ経由でのキャプチャが重くなる可能性。
- **プライバシー:** パスワード等の伏せ字処理が必須。

## 4. Act: 実装フェーズへの移行
- **UIコンセプト:** 「AI感のない」洗練されたダークテーマ。VS Codeのようなエンジニア向けかつ、経営層も安心できる堅牢なデザイン。
- **プロトタイプ構成:**
  - `Dashboard`: 進行中のエージェントタスク一覧。
  - `Shadow Log`: 操作のタイムライン。
  - `Approval Request`: 人間の判断を仰ぐポップアップ。
