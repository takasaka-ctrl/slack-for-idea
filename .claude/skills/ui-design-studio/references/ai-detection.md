# AI感検出・排除ガイド

AI生成デザインの典型的パターンを検出・排除するための統合ガイド。

---

## AI感とは

「AI感」「AI Slop」とは、AI生成コンテンツに見られる**没個性的で画一的な特徴**。

### なぜ問題か

1. **差別化の欠如**: 他のAI生成物と見分けがつかない
2. **ブランドアイデンティティの希薄化**: 独自の個性が表現できない
3. **信頼性の低下**: 「手抜き」「安っぽい」という印象
4. **記憶に残らない**: 見た瞬間に忘れられる

### 根本原因

AIは学習データの**平均的なパターン**に収束する傾向がある：
- 最も一般的なデザインパターンを再現
- 「安全」で「無難」な選択に偏る

## Critical（該当したら修正）

以下に該当する場合、**正当な理由がなければ修正必須**。

### タイポグラフィ

| パターン | 判定 | 許容されるケース |
|----------|------|-----------------|
| Inter単独使用（見出し・本文両方） | 修正 | 本文のみ＋見出しに個性的フォント |
| Roboto単独使用 | 修正 | Googleプロダクト連携を意図 |
| Arial/Helvetica単独使用 | 修正 | ブランドガイドライン準拠 |
| system-ui単独使用 | 修正 | パフォーマンス最優先 |
| フォント1種類のみ | 修正 | Ultra Minimalで意図的 |

**改善例**:
```css
/* Before（AI感あり） */
font-family: Inter, sans-serif;

/* After（個性あり） */
/* 見出し */ font-family: "Space Grotesk", sans-serif;
/* 本文 */   font-family: "DM Sans", sans-serif;
```

### カラー

| パターン | 判定 | 許容されるケース |
|----------|------|-----------------|
| 白背景 + 紫→青グラデーション | 修正 | AI/テック企業で意図的、またはパロディ |
| blue-500単独アクセント | 修正 | 企業ブランドカラー |
| 彩度100%原色のみ | 修正 | Neo Brutalismで意図的 |

**改善例**:
```css
/* Before（AI感あり） */
background: linear-gradient(135deg, #8b5cf6, #3b82f6);

/* After（個性あり） */
background: #0f172a;  /* ソリッドカラー */
/* または独自のグラデーション */
background: linear-gradient(135deg, #1e3a5f 0%, #0d1b2a 100%);
```

### レイアウト

| パターン | 判定 | 許容されるケース |
|----------|------|-----------------|
| 完全対称3カラムカード | 修正 | **日本市場LP/比較表/料金プラン** |
| 左テキスト・右画像ヒーロー | 修正 | 製品デモを見せる必要 |
| 全要素同一border-radius | 修正 | デザインシステム統一 |

**改善例**:
```html
<!-- Before（AI感あり） -->
<div class="grid grid-cols-3 gap-4">
  <Card /><Card /><Card />
</div>

<!-- After（個性あり）- Bento Grid -->
<div class="grid grid-cols-4 grid-rows-2 gap-4">
  <Card class="col-span-2 row-span-2" />
  <Card />
  <Card />
  <Card class="col-span-2" />
</div>
```

### コンテンツ

| パターン | 判定 | 許容されるケース |
|----------|------|-----------------|
| 「革新的」「シームレス」「次世代」 | 常に修正 | なし |
| 「今日の急速に変化する世界で」 | 常に修正 | なし |
| Lorem ipsum放置 | 常に修正 | プロトタイプ明示のみ |

## Warning（改善を検討）

### タイポグラフィ
- [ ] Open Sans / Lato / Montserrat 単独使用
- [ ] フォントウェイトが2種類以下
- [ ] letter-spacing調整なし

### カラー
- [ ] Tailwindデフォルトカラーそのまま
- [ ] アクセントカラー1色のみ
- [ ] 背景が純白（#ffffff）のみ

### レイアウト
- [ ] 余白が均一すぎる
- [ ] カードシャドウがデフォルト（shadow-md等）
- [ ] ホバーエフェクトなし

### コンテンツ
- [ ] CTA文言が「Get Started」「Learn More」のみ
- [ ] 汎用アイコンセットのデフォルト使用

---

## Must Have（必須要素）

良いデザインには以下が含まれる：

### 明確な美的方向性
- [ ] スタイル名が言える（例：Swiss Minimal, Neo Brutalism）
- [ ] なぜそのスタイルを選んだか説明できる
- [ ] 参照したデザインがある

### 記憶に残る要素
- [ ] 1つ以上のユニークな視覚要素
- [ ] 予想外のデザイン選択が1つ以上
- [ ] コンテキストに応じたカスタマイズ

### カラーパレットの意図
- [ ] 3-5色の制限されたパレット
- [ ] 色の選択理由が説明できる
- [ ] 60-30-10ルールに近い配分

### タイポグラフィの個性
- [ ] 見出しと本文で異なるフォント
- [ ] 適切なフォントウェイトの使い分け
- [ ] 文脈に合ったフォント選択

## 独自性を出す戦略

### 1. 明確な美的方向性

汎用的な「モダンでクリーンな」ではなく、具体的なスタイルを選択：

- **ブルータリスト**: 生々しく、装飾を排除
- **ネオブルータリズム**: 高コントラスト、ブロック型、太字
- **マキシマリスト**: 密度高く、情報過多を美として
- **レトロフューチャリスト**: 過去のSFビジョン
- **オーガニック**: 自然の形態、曲線
- **エディトリアル**: 雑誌のような構成
- **アールデコ**: 幾何学的、装飾的

### 2. 具体性の追加

- **実際のデータ**: プレースホルダーではなくリアルな内容
- **文脈のあるビジュアル**: 抽象的でない具体的なイメージ
- **ブランドの声**: 独自のトーン&マナー

### 3. 制約の活用

- **限定されたカラーパレット**: 2-3色で勝負
- **単一フォントファミリー**: ウェイトで変化
- **ミニマルな要素**: 少ないほど独自性が際立つ

## 市場別ガイドライン

### グローバル/欧米市場

**厳格に適用**:
- 3カラム対称レイアウトは避ける
- 紫青グラデーションは避ける
- 情報密度は抑えめ

### 日本市場

**許容される例外**:
- 3カラム対称レイアウト（比較表、料金プラン）
- 高い情報密度（信頼バッジ、FAQ大量）
- 電話番号の目立つ配置

**厳格に適用**:
- 空虚なコピーは避ける（「革新的」等）
- 汎用イラストは避ける

## スコアリング

### 計算方法

```
base_score = 10

# Critical違反: -2点/each
# Warning: -0.5点/each
# Must Have不足: -1点/each missing

final_score = max(0, base_score - violations)
```

### スコア解釈

| スコア | 評価 | アクション |
|--------|------|-----------|
| 9-10 | 優秀 | 個性的で記憶に残る |
| 7-8 | 良好 | 小さな改善で優秀に |
| 5-6 | 普通 | AI感が残る、要改善 |
| 3-4 | 要注意 | 大幅な見直し必要 |
| 0-2 | 不可 | 根本的な再設計必要 |

## クイックチェック（7つの質問）

以下に全て「はい」と答えられるか：

1. このデザインを見て、どのスタイル/ジャンルか説明できるか？
2. 他のAI生成サイトと見分けがつくか？
3. 1週間後に思い出せる要素があるか？
4. なぜこのフォントを選んだか説明できるか？
5. なぜこのカラーを選んだか説明できるか？
6. ターゲットユーザーにとって適切か？
7. ブランドの個性が表現されているか？

## 正当化（Justification）の書き方

Criticalパターンに該当するが許容する場合：

```yaml
ai_detection:
  violations:
    - pattern: "3カラム対称レイアウト"
      status: "JUSTIFIED"
      justification: |
        日本市場向けLPであり、料金プランの比較表として
        3カラム対称が最も情報を比較しやすい。
        japan-market.md のガイドラインに準拠。
  unjustified_count: 0
  overall: "PASS"
```
