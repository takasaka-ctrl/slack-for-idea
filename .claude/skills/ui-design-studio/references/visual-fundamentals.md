## 余白（ホワイトスペース）

### 余白の種類

#### マクロホワイトスペース
コンテンツブロック間、セクション間の大きな余白
- **目的**: コンテンツの整理、ユーザーの誘導、呼吸感の創出
- **効果**: 洗練された印象、ラグジュアリー感

#### マイクロホワイトスペース
行間、文字間、要素内部の小さな余白
- **目的**: 可読性の向上、視認性の改善
- **効果**: テキストの読みやすさ、UIの使いやすさ

### 余白の効果

| 余白の量 | 印象 | 適した用途 |
|---------|------|-----------|
| 多い | ラグジュアリー、ミニマル | 高級ブランド、アート |
| 適度 | バランス、プロフェッショナル | 企業サイト、SaaS |
| 少ない | 情報密度高、ニュース的 | メディア、EC |

### 余白のベストプラクティス

1. **近接の原則に従う**
   - 関連要素は近くに（8-16px）
   - 無関係要素は遠くに（24-48px以上）
   - 曖昧な距離を避ける

2. **一貫したスペーシングシステム**
   - 8pxグリッドベース: 8, 16, 24, 32, 48, 64, 96
   - または4pxグリッド: 4, 8, 12, 16, 24, 32, 48
   - ランダムな値を避ける

3. **重要な要素を余白で強調**
   - 周囲の余白が多いほど注目度UP
   - CTAの周りは十分な余白を確保

4. **コンテンツ幅の制限**
   - 本文: 65-75文字（45-75chが理想）
   - 広すぎると読みにくい

## 色の使い方

### カラーパレットの構築

#### 基本構成
- **プライマリカラー**: ブランドを代表する色（1色）
- **セカンダリカラー**: 補助的な色（1-2色）
- **アクセントカラー**: 強調用（1色）
- **ニュートラル**: 背景、テキスト用グレースケール（5-7段階）
- **セマンティック**: 成功（緑）、警告（黄）、エラー（赤）、情報（青）

#### カラースキームの種類
- **類似色**: 隣り合う色の組み合わせ（調和）
- **補色**: 対角線上の色（高コントラスト）
- **トライアド**: 等間隔の3色（バランス）
- **モノクロマティック**: 単色の濃淡

### 色のコントラスト

#### WCAG基準
- 通常テキスト（18pt未満）: 4.5:1以上
- 大きいテキスト（18pt以上/14pt太字以上）: 3:1以上
- UI要素: 3:1以上

#### チェック方法
- Chrome DevToolsのアクセシビリティ監査
- WebAIM Contrast Checker
- Figmaプラグイン

### 色の心理的効果

| 色 | 印象 | 適した業種 |
|----|------|-----------|
| 青 | 信頼、安定、専門性 | 金融、テック、医療 |
| 緑 | 自然、成長、健康 | 環境、ウェルネス、金融 |
| 赤 | 情熱、緊急、エネルギー | 飲食、エンタメ、セール |
| 黄/オレンジ | 楽観、創造性、注意 | 子供向け、創造系 |
| 紫 | 高級、創造性、神秘 | 美容、ラグジュアリー |
| 黒 | 高級、洗練、力強さ | ファッション、ラグジュアリー |

### 色のベストプラクティス

1. **60-30-10ルール**
   - 60%: 支配色（背景など）
   - 30%: 副色（セクション、カードなど）
   - 10%: アクセント（CTA、リンクなど）

2. **アクセントカラーは限定的に**
   - 乱用すると効果が薄れる
   - CTAや重要なアクションのみ

3. **色だけに頼らない**
   - アイコン、テキスト、形状でも情報を伝達
   - 色覚多様性への配慮

## タイポグラフィ

### タイプスケール

基準サイズ（16px）からの倍率で設計：

| 用途 | サイズ | 倍率 |
|------|-------|------|
| H1 | 48px | 3.0x |
| H2 | 36px | 2.25x |
| H3 | 24px | 1.5x |
| H4 | 20px | 1.25x |
| Body | 16px | 1.0x |
| Small | 14px | 0.875x |
| Caption | 12px | 0.75x |

### 行間（Line Height）

- 見出し: 1.1-1.3
- 本文: 1.5-1.7（120-170%）
- 行間が狭すぎると読みにくい
- 広すぎると文章のまとまりが失われる

### 文字間（Letter Spacing）

- 見出し（大文字）: 少し広げる（0.05-0.1em）
- 本文: デフォルトか微調整
- 小さいテキスト: わずかに広げる

### フォントペアリングの原則

1. **コントラストを作る**
   - Serif + Sans-serif
   - Display + Text
   - 異なるx-height

2. **最大3フォントまで**
   - 見出し用
   - 本文用
   - （必要なら）アクセント用

3. **同じファミリーのウェイト活用**
   - Light, Regular, Medium, Bold
   - フォント数を増やさず変化を出す

### 推奨フォントペアリング

| 見出し | 本文 | 印象 |
|--------|------|------|
| Playfair Display | Lato | エレガント、編集的 |
| Oswald | Source Sans Pro | モダン、シャープ |
| Montserrat | Merriweather | 現代的、読みやすい |
| Space Grotesk | Inter | テック、クリーン |
| Cormorant | Nunito | ラグジュアリー、柔らか |

## ボタンとCTA

### ボタンの階層

| タイプ | 用途 | 視覚的処理 |
|--------|------|-----------|
| Primary | 主要アクション | 塗り、高コントラスト |
| Secondary | 補助アクション | アウトライン or 薄い塗り |
| Tertiary | その他のアクション | テキストのみ or ゴースト |

### ボタンサイズ

- **大（Desktop CTA）**: 高さ48-56px、パディング24-32px
- **中（通常）**: 高さ40-44px、パディング16-24px
- **小**: 高さ32-36px、パディング12-16px
- **最小タッチターゲット**: 44×44px（Apple）/ 48×48px（Material）

### CTAの配置

#### 効果的な配置位置
- **Above the fold**: スクロールなしで見える位置
- **コンテンツの終点**: 情報を読んだ後
- **F/Zパターンの視線上**: 自然に目に入る位置

#### F/Zパターン
- **Fパターン**: 情報量の多いページ（ブログ等）
- **Zパターン**: シンプルなランディングページ

### CTAのマイクロコピー

**効果的なフレーズ**:
- 動詞で開始: 「始める」「入手する」「試す」
- 具体的な価値: 「無料で試す」「30日間無料」
- 1人称: 「私の席を確保」「マイページへ」

**避けるべきフレーズ**:
- 「送信」「Submit」
- 「クリックここ」
- 曖昧な「詳しく見る」

### ボタン状態

必要な状態を定義：
- Default（通常）
- Hover（ホバー）
- Active/Pressed（押下中）
- Focus（フォーカス）
- Disabled（無効）
- Loading（読み込み中）

## レイアウトと構成

### グリッドシステム

#### 12カラムグリッド
- 柔軟な分割: 1, 2, 3, 4, 6, 12
- 一般的な構成: コンテンツ(8) + サイドバー(4)
- ガター: 16-32px

#### マージンとパディング
- モバイル: 16-24px左右マージン
- タブレット: 32-48px
- デスクトップ: 64px以上または最大幅制限

### 視覚的階層の作り方

1. **サイズ**: 重要なものを大きく
2. **色/コントラスト**: 重要なものを目立たせる
3. **位置**: 上部・中央に重要なもの
4. **余白**: 周囲の空間で強調
5. **タイポグラフィ**: ウェイトで差別化

### 構成パターン

#### ヒーローセクション
- 大きな見出し + サブ見出し
- 主要CTA（+ 補助CTA）
- ビジュアル要素

#### 特徴/ベネフィットセクション
- 3-4カラムのグリッド
- アイコン + 見出し + 説明
- または交互配置（画像-テキスト）

#### 社会的証明
- ロゴ群（導入企業）
- テスティモニアル（顔写真 + 引用）
- 数字の強調（ユーザー数、実績）

#### 価格表
- 3プラン比較が理想
- 推奨プランを視覚的に強調
- 機能比較表

### レスポンシブデザイン

#### ブレイクポイント例
```css
/* Mobile First */
/* デフォルト: 〜639px */
/* sm: 640px〜 */
/* md: 768px〜 */
/* lg: 1024px〜 */
/* xl: 1280px〜 */
/* 2xl: 1536px〜 */
```

#### 切り替えポイントでの調整
- カラム数の変更
- フォントサイズの調整
- ナビゲーションの変更（ハンバーガー）
- 余白の調整

| 項目 | 推奨値 |
|------|--------|
| 本文フォントサイズ | 16px |
| 行間（本文） | 1.5-1.7 |
| 最大行長 | 65-75文字 |
| 最小タッチターゲット | 44-48px |
| スペーシング基準 | 8pxグリッド |
