# TruthLens - AI Fact-Check Browser Extension - PDCA評価

## Plan: 問題起点のアイデア生成

### 背景問題
- 2026年、生成AIコンテンツの爆発的増加により誤情報が氾濫
- ソーシャルメディアでの拡散が速すぎ、ファクトチェックが追いつかない
- 既存ツールは別タブ確認など摩擦が大きく、利用率が低い

### アイデア候補

#### 1. TruthLens (選択)
**概要**: リアルタイムで記事・SNS投稿の信頼性を評価するブラウザ拡張
**差別化**: インライン表示、主張の分解、ソース分析、バイアス検出
**実装**: Chrome/Firefox Extension + OpenAI API + 信頼性DB

#### 2. NewsChain
**概要**: ブロックチェーンで記事の出自と編集履歴を記録
**課題**: 既存メディアの協力が必須、採用ハードルが高い

#### 3. CrowdVerify
**概要**: 専門家コミュニティによる分散型ファクトチェック
**課題**: コールドスタート問題、専門家の招集が困難

#### 4. SourceGraph
**概要**: 情報源の信頼性ネットワークを可視化
**課題**: ニッチすぎる、一般ユーザーには複雑

**選択理由**: TruthLensは技術的実現可能性が高く、UXの摩擦が最小で、即座に価値を提供できる。

## Check: ギャップ分析

### 強み
- 極めて高い問題重要性（9.0）と感情的インパクト（9.0）
- 巨大市場、完璧なタイミング（選挙年、AI規制）
- 明確な差別化（インライン表示、主張分解）

### 弱点
1. **実装の複雑さ**: AI精度とFalse Positiveのバランスが難しい
2. **競合優位性**: モートが弱い（6.7）、データベース以外の防御策が必要
3. **信頼性の確保**: 誤った判定が信頼を損なうリスク

### 改善優先度
1. **高**: モート強化（コミュニティフィードバック、学習ループ）
2. **中**: 実装容易性向上（MVP範囲の縮小、段階的リリース）
3. **低**: False Positiveリスク管理（保守的な判定、透明性）

## Act: アイデア改善

### 改善版: TruthLens v2

#### 追加機能
1. **コミュニティフィードバックループ**
   - ユーザーが判定に同意/不同意を投票
   - 専門家の検証結果を学習データに追加
   - → ネットワーク効果によるモート構築

2. **段階的信頼性表示**
   - 過激な判定を避け、「要確認」「検証中」「信頼性高」の3段階
   - False Positiveリスクを最小化

3. **透明性ダッシュボード**
   - 判定理由を詳細表示（引用元、類似記事、専門家意見）
   - アルゴリズムのバイアスを開示

4. **専門家ネットワーク連携**
   - 医療、法律、科学などの分野別専門家の検証を優先表示
   - → 信頼性とクオリティのモート

- **Enterprise** ($49/ユーザー/月): API、カスタムルール、チーム管理

#### 成長戦略
1. **Phase 1**: Chrome拡張リリース、X/Reddit重点対応

## プロトタイプ仕様

### MVP範囲（v0.1）
1. **対応プラットフォーム**: X (Twitter)のみ
2. **機能**:
   - 投稿の横に信頼性バッジ表示（🟢🟡🔴）
   - クリックで詳細パネル展開
   - 主張の分解と根拠リンク
   - ソース分析（リンク先の信頼性評価）

3. **技術スタック**:
   - Chrome Manifest V3 Extension
   - Content Script（DOM監視、バッジ挿入）
   - Background Service Worker（API通信）
   - モックAI判定（初期版はルールベース）

4. **デザイン原則**:
   - 非侵襲的（既存UIを邪魔しない）
   - プロフェッショナル（Notion/Linear風）
   - 透明性（判定理由を明示）
   - AI感を排除（グラデーション・グローなし）

### ファイル構成
```
truthlens/
├── manifest.json       - Extension設定
├── content.js          - DOM監視、バッジ挿入
├── background.js       - API通信、状態管理
├── popup.html/js/css   - 拡張アイコンクリック時のUI
├── styles.css          - インライン要素スタイル
├── trust-engine.js     - 信頼性判定ロジック（モック）
└── README.md
```
