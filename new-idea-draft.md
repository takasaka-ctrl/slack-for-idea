# 新規アイデア候補: Focus Flow

## 問題起点

**課題:**
リモートワーク・フリーランスの深い作業（ディープワーク）が、頻繁な通知・メッセージ・電話で分断される。
- Slackの「後で見る」は見ない
- 集中したいときに限って緊急っぽい連絡が来る
- 手動でスマホを機内モードにしても、本当に緊急の連絡を逃す

**ターゲット:**
- リモートワーカー（エンジニア、デザイナー、ライター）
- フリーランス
- 経営者・管理職（会議と深い思考の両立が必要）

---

## アイデア候補（3案）

### 案1: AI秘書型 Focus Flow
**コンセプト:** 深い作業中の割り込みをAIが自動判断して対応

**機能:**
- ポモドーロタイマー + AI秘書
- 集中モード中の連絡を全てAIが受信
- 緊急度を判断（例: 顧客 > 同僚、締切関連 > 雑談）
- 緊急でなければ「今集中中、◯時に返信します」と自動返信
- 本当に緊急なら通知 or AI音声で割り込み
- 休憩時間に要約レポート提示

### 案2: チーム連携型 Focus Sync
**コンセプト:** チーム全体の集中時間を可視化・調整

**機能:**
- カレンダー + Slack統合
- チームメンバーの「集中ブロック」を共有
- 「今誰が集中中か」をリアルタイム表示
- 緊急連絡したいとき、代わりに対応できる人を提案
- 集中時間のアナリティクス（週◯時間確保できたか）

### 案3: 環境連動型 Deep Work Hub
**コンセプト:** 物理環境 + デジタルを統合制御

**機能:**
- スマート照明・音響と連動（集中モードで照明変化、ノイズキャンセル音）
- スマホ・PCの通知を一元管理
- VR/AR対応（仮想ワークスペース）
- バイオメトリクス（心拍・集中度）でモード自動調整
- 家族・同居人への「集中中サイン」投影

---

## PDCA評価

### Plan: 案1「AI秘書型 Focus Flow」を詳細化

**理由:**
- 実装が現実的（API統合中心）
- 個人ユーザーですぐ使える
- 案2はチーム導入ハードル高、案3はハードウェア依存

**詳細設計:**
1. **集中セッション開始**
   - ユーザーが「2時間 集中モード」を設定
   - 目的を入力（例: 「プレゼン資料作成」）

2. **AI自動対応**
   - Slack, Email, 電話, SMS を監視
   - LLMで内容 + 送信者の重要度を判断
   - 緊急度スコア算出（0-10）

3. **対応フロー**
   - スコア 0-3: 自動で「後で返信します」
   - スコア 4-6: 音声アラート「◯◯さんから連絡、確認しますか？」
   - スコア 7-10: 即座に通知 + 概要読み上げ

4. **休憩時レポート**
   - 受信メッセージ一覧
   - 優先度順に並び替え
   - 推奨返信文を生成

**技術スタック:**
- フロントエンド: React/Next.js PWA
- バックエンド: Node.js + LLM API (Claude/GPT)
- 統合: Slack API, Gmail API, Twilio (電話)
- 音声: ElevenLabs TTS

### Check: ギャップ分析

**強み:**
- 明確な課題解決（集中の分断）
- 高い有用性・満足度
- 成長市場

**弱点:**
1. **技術優位性が弱い（6.5）**
   - API統合は誰でもできる
   - 大手参入で埋もれるリスク

2. **緊急度判断の難しさ**
   - AI誤判定が致命的
   - 学習データ収集に時間

3. **プライバシー懸念**
   - メール・メッセージの内容解析 → ユーザー不安

### Act: 改善案

#### 改善1: 「学習型パーソナルAI」で技術優位性強化

**追加:**
- 最初はユーザーが手動で緊急度を修正
- その修正をAIが学習し、個人特化
- 使うほど賢くなる → 乗り換えコスト増
- 「1ヶ月使えば完璧」を訴求

**効果:**
- Advantage: 6.5 → **7.5** (+1.0)
- Confidence: 7.5 → **8.0** (+0.5)

#### 改善2: プライバシー・ファースト設計

**追加:**
- **完全ローカル処理オプション**（オンデバイスLLM）
- メッセージ内容は一切クラウド送信しない
- 送信者・件名・キーワードのみで判断
- ユーザーが選べる（クラウドAI vs ローカルAI）

**効果:**
- Confidence: 8.0 → **8.5** (+0.5)
- Ease: 7.0 → **7.5** (+0.5)

#### 改善3: チーム機能で差別化

**追加:**
- 「誰が今集中中か」を可視化
- 緊急時に「代わりに対応できる人」を提案
- チーム全体の集中時間アナリティクス

**効果:**
- Advantage (ネットワーク効果): 5.0 → **7.0** (+2.0)
- Market: 7.83 → **8.2** (+0.37、企業需要増）
